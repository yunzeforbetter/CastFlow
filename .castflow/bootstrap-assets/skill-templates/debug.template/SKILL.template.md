# Debug-Skill - 边界安全巡检

**定位**: 质量守卫 (Quality Guard)，系统地验证代码在边界场景下的正确性。

**使用方式**:
- **直接使用**: "用 debug-skill 检查这段代码的边界条件"
- **Agent 加载**: 任何 agent 都可以在需要时加载此 skill
- **Pipeline 编排**: code-pipeline 在 Step 7 中调用

**核心职责**:
1. **边界压力测试**: 识别 null/empty/out-of-range 情况
2. **状态一致性核查**: 检查生命周期清理、并发冲突
3. **集成安全验证**: 验证外联依赖、配置表的防御性
4. **决策反馈**: 提供包含修复建议的报告

## 调试决策参数 (L1)

在启动前需确定检查深度：

- **strictness**:
  - Deep: 检查全部清单项（核心逻辑必选）
  - Focused: 重点核查数据边界和状态转换（常规业务）
  - Quick: 仅检查 P0 级 Null/Empty 防护（外围功能）

## 边界检查清单

### Category 1: 数据边界
- [ ] 1.1 **空引用** - 所有对象成员访问都有 null 检查
- [ ] 1.2 **空集合** - 遍历前检查 Count > 0
- [ ] 1.3 **数组越界** - 索引必须处于 [0, Count-1]
- [ ] 1.4 **数值溢出** - 数学操作有 Min/Max 限制
- [ ] 1.5 **字符串边界** - 空字符串校验
- [ ] 1.6 **默认值混淆** - 区分 0/null 与有效值的逻辑
- [ ] 1.7 **字典查询** - 使用安全的查询方式
- [ ] 1.8 **枚举遍历** - 处理了空列表或超大列表场景
- [ ] 1.9 **类型转换** - 类型转换后都有 null/类型检查
- [ ] 1.10 **资源容量** - 内存或队列设置了上限

### Category 2: 状态转换
- [ ] 2.1 **前提条件** - 方法开始前确认初始化状态
- [ ] 2.2 **状态清理** - 关闭/销毁时释放资源
- [ ] 2.3 **对象销毁** - 避免访问已销毁的对象
- [ ] 2.4 **并发锁** - 多线程访问的状态有保护
- [ ] 2.5 **原子性** - 关键流程"要么全成功，要么全失败"
- [ ] 2.6 **部分初始化** - 防止访问未分配字段
- [ ] 2.7 **枚举校验** - 对外部输入的枚举值做范围检查
- [ ] 2.8 **顺序约束** - Init -> Execute -> Close 顺序规范化

### Category 3: 集成边界
- [ ] 3.1 **API 契约** - 参数符合接口签名且已校验
- [ ] 3.2 **异步超时** - 异步操作必须有超时处理
- [ ] 3.3 **配置验证** - 读取配置项前验证存在性
- [ ] 3.4 **循环依赖** - 禁止事件处理中触发自身事件
- [ ] 3.5 **缓存失效** - 源数据更新后缓存是否同步失效
- [ ] 3.6 **版本兼容** - 本地数据读取旧版本的兼容性
